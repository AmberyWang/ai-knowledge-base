# Shadowrocket 手机端配置静态住宅IP指南 (链式代理)

本文档介绍如何在手机端 (iOS Shadowrocket / 小火箭) 配置 **静态住宅 IP**，并通过机场节点进行 **链式中转** (Proxy via)，实现由机场节点自动容灾加速连接住宅 IP 的效果。

## 适用场景


- 需要在手机上使用纯净住宅 IP 访问 Claude, ChatGPT 等高风控服务。
- 直接连接住宅 IP 速度慢或被墙，需要通过机场节点中转加速。

## 前置准备


1. **静态住宅 IP 信息**：IP 地址、端口、用户名、密码。
2. **Shadowrocket App**：已安装并配置好常用的机场订阅。


---

## 步骤一：手动添加“静态住宅 IP”节点


1. 打开 **Shadowrocket**。
2. 点击右上角的 **`+`** 号。
3. **类型 (Type)** 选择：`HTTP` (或 SOCKS5，视您购买的 IP 类型而定)。
4. 填写您的住宅 IP 信息：

  - **地址 (Address)**: `168.158.198.151` (示例)
  - **端口 (Port)**: `12323`
  - **认证 (Authentication)**: 开启
  - **用户 (User)**: `your_username`
  - **密码 (Password)**: `your_password`
  - **备注 (Remarks)**: `🇺🇸 美国静态住宅`

5. 点击右上角 **保存**。

此时，您会在节点列表底部看到这个新添加的节点。**暂时不要选中它**，因为它现在是直连的，速度可能很慢或无法连接。


---

## 步骤二：配置“代理通过” (Proxy Via) 实现链式中转

这一步的核心是让这个住宅节点**不直连**，而是强制通过您的机场节点（自动优选组）进行中转。


1. 在 Shadowrocket 首页节点列表中，找到刚才添加的 **`🇺🇸 美国静态住宅`**。
2. 点击该节点右侧的 **`i`**** (信息图标)** 进入编辑模式。
3. 向下滑动，找到 **`代理通过 (Proxy Via)`** 选项（默认是 `无/None`）。
4. 点击它，在弹出的列表中选择您的 **机场自动优选策略组**（通常叫 `Proxy`、`自动选择` 或 `Url-Test` 组）。

  - *原理：流量 -> 手机 ->  ->  -> 目标网站*
  - *优势：机场组会自动选择最快的线路，实现容灾和加速。*

5. 点击右上角 **保存**。


---

## 步骤三：验证与使用


1. 回到 Shadowrocket 首页。
2. **全局路由** 建议选择 `配置 (Config)`。
3. 在节点列表中，选中 **`🇺🇸 美国静态住宅`**。
4. 开启顶部的连接开关。
5. **验证 IP**：

  - 使用浏览器访问 (https://ipinfo.io "ipinfo.io") 或 (https://whoer.net "whoer.net")。
  - 如果显示的 IP 是您的 **静态住宅 IP** (如 168.158.xxx.xxx)，且 ISP 显示为住宅代理商（而非机场名），说明配置成功！


## 常见问题


- **Q: 为什么连不上？** A: 请检查机场节点是否正常，或者住宅 IP 是否过期。也可以尝试将 `代理通过` 换成一个具体的机场节点测试。
- **Q: 速度慢？** A: 住宅 IP 本身带宽通常有限，且经过了两次跳转，延迟会比直接用机场高，这是正常的。
