## 一、项目概述

### 1.1 项目定位

Oh-My-OpenCode 是一个 **OpenCode 插件**，为 Claude Code 提供多模型 AI 代理编排能力。它类似于 "oh-my-zsh" 之于 zsh 的关系——提供开箱即用的增强功能和高度可定制性。

### 1.2 核心能力


| 能力类别 | 描述 |
| --- | --- |
| **多模型编排** | 支持 Claude Opus 4.5、GPT-5.2、Gemini 3、Grok、GLM-4.7 等多种模型协同工作 |
| **AI 代理系统** | 10 个专业化代理，各司其职，协同完成复杂任务 |
| **生命周期钩子** | 31 个钩子拦截和修改代理行为 |
| **增强工具集** | 20+ 工具（LSP、AST-Grep、会话管理、代理委托等） |
| **后台任务** | 并行执行多个代理，提升效率 |
| **Claude Code 兼容** | 完整兼容层，支持 Claude Code 的命令、技能、代理、MCP、钩子 |

### 3.3 后台任务管理器

#### 3.3.1 核心类 `BackgroundAgentManager`

```TypeScript
// 代码块
// src/features/background-agent/manager.ts (1336行)
export class BackgroundAgentManager {
  private tasks: Map<string, BackgroundTask> = new Map()
  private concurrencyLimits: ConcurrencyConfig
  
  // 启动后台任务
  async startTask(config: TaskConfig): Promise<string> {
    const taskId = generateTaskId()
    const task = new BackgroundTask(config)
    
    // 并发控制
    await this.acquireConcurrencySlot(config.model)
    
    // 异步执行
    task.execute().then(() => {
      this.releaseConcurrencySlot(config.model)
      this.notifyCompletion(taskId)
    })
    
    this.tasks.set(taskId, task)
    return taskId
  }
  
  // 获取任务输出
  async getOutput(taskId: string, options?: OutputOptions): Promise<TaskOutput> {
    const task = this.tasks.get(taskId)
    if (!task) throw new TaskNotFoundError(taskId)
    
    if (options?.block) {
      await task.waitForCompletion(options.timeout)
    }
    
    return task.getOutput()
  }
  
  // 取消任务
  async cancelTask(taskId: string): Promise<void> {
    const task = this.tasks.get(taskId)
    task?.cancel()
  }
}

```

```TypeScript
// 代码块
// src/features/background-agent/manager.ts - 简化版核心逻辑
export class BackgroundAgentManager {
  private tasks = new Map<string, BackgroundTask>()
  private concurrencySlots = new Map<string, number>()
  
  async startTask(config: TaskConfig): Promise<TaskStartResult> {
    const taskId = `bg_${crypto.randomUUID().slice(0, 8)}`
    
    // 并发检查
    const provider = getProvider(config.model)
    const limit = this.getConcurrencyLimit(provider)
    const current = this.concurrencySlots.get(provider) || 0
    
    if (current >= limit) {
      return { taskId, status: "queued", message: "等待并发槽位" }
    }
    
    // 创建任务
    const task = new BackgroundTask({
      id: taskId,
      agent: config.agent,
      prompt: config.prompt,
      model: config.model,
    })
    
    // 增加并发计数
    this.concurrencySlots.set(provider, current + 1)
    this.tasks.set(taskId, task)
    
    // 异步执行
    task.execute()
      .finally(() => {
        // 释放并发槽位
        const count = this.concurrencySlots.get(provider) || 1
        this.concurrencySlots.set(provider, count - 1)
        // 触发通知
        this.notifyCompletion(taskId)
      })
    
    return { taskId, status: "running" }
  }
  
  async getOutput(taskId: string, options?: { block?: boolean; timeout?: number }) {
    const task = this.tasks.get(taskId)
    if (!task) throw new Error(`Task ${taskId} not found`)
    
    if (options?.block && task.status === "running") {
      await task.waitForCompletion(options.timeout)
    }
    
    return {
      taskId,
      status: task.status,
      output: task.output,
      duration: task.duration,
    }
  }
}
```

## 八、关键设计理念

### 8.1 "巨石推动者" 哲学

Sisyphus 代理的设计理念来源于希腊神话中的西西弗斯——永不停歇地推动巨石。在这个项目中：


- **TODO 列表是巨石**：代理必须完成所有 TODO 项目才能停止
- **TODO 延续钩子是惩罚**：如果代理试图中途放弃，钩子会强制其继续
- **多代理协作是工具**：代理可以委托任务，但最终责任在 Sisyphus

### 8.2 专业化分工

每个代理都有明确的专业领域：


- **Sisyphus**: 总指挥，负责任务分解和协调
- **Oracle**: 战略顾问，处理高难度架构和调试
- **Librarian**: 文档专家，搜索外部资源
- **Explore**: 侦察兵，快速扫描代码库
- **Frontend Engineer**: 前端专家，处理 UI/UX

### 8.3 安全隔离

通过工具权限控制实现安全隔离：


- Oracle 只能读取，不能写入
- Explore 只能搜索，不能修改
- 只有主代理 Sisyphus 有完整权限

## 九、总结

Oh-My-OpenCode 通过以下核心机制实现了强大的 AI 代理编排能力：


1. **多模型协同**: 不同模型负责不同领域，发挥各自优势
2. **生命周期钩子**: 31 个钩子提供全面的行为控制点
3. **后台并行**: 支持多任务并行执行，提升效率
4. **工具增强**: LSP、AST-Grep 等工具提供精确的代码操作能力
5. **安全隔离**: 通过权限控制保证系统安全
6. **高度可配置**: 几乎所有行为都可以通过配置调整

这个项目展示了如何构建一个生产级的 AI 代理编排系统，值得深入学习和参考。
